# Бот-копирайтер — шаги для тебя (урок М4)

Код уже лежит в папке **`copywriter-bot`**. Его нужно **запустить у себя** и **подставить свои секреты** — я не могу сделать это за тебя (токены только у тебя).

---

## Шаг 1. Создать бота в Telegram

1. Открой Telegram → **@BotFather** → `/newbot` (или возьми уже созданного бота).  
2. Сохрани **токен** (длинная строка) — это `TELEGRAM_BOT_TOKEN`.

---

## Шаг 2. Ключ API нейросети

Нужен любой **OpenAI-совместимый** чат-API.

- **OpenAI:** https://platform.openai.com/api-keys → ключ вида `sk-...`  
- **OpenRouter** (много моделей): https://openrouter.ai/keys → ключ `sk-or-v1-...`, в `.env` тогда другой `CHAT_API_URL` (см. `.env.example`).

Без денег на API бот **не сможет** вызывать нейросеть (локальные модели в этой версии не подключены).

---

## Шаг 3. Файл стиля

В папке уже есть **`tone-of-voice.md`** (копия с урока 2).  
Можешь **заменить** его своим файлом из папки курса или отредактировать под себя.

Команда **`/reloadstyle`** перечитывает файл с диска без перезапуска.

---

## Шаг 4. Настроить `.env`

1. Скопируй **`copywriter-bot/.env.example`** → **`copywriter-bot/.env`**.  
2. Вставь `TELEGRAM_BOT_TOKEN` и `CHAT_API_KEY`.  
3. При необходимости поправь `CHAT_MODEL` и `CHAT_API_URL`.

---

## Шаг 5. Установить зависимости и запуск

В PowerShell:

```text
cd C:\Users\HP\Documents\Projects\my-first-project\copywriter-bot
npm install
npm start
```

Окно **не закрывай** — пока оно открыто, бот онлайн.

Открой своего бота в Telegram и напиши, например:  
*«Напиши короткий пост: зачем эксперту лендинг, а не огромный сайт»*.

---

## Мультиагентность (как в уроке, упрощённо)

В `.env` поставь **`USE_MULTISTEP=1`** — тогда сначала идёт запрос «аналитик» (короткий план), потом «копирайтер» (пост).  
Это **2 вызова API** (дороже, дольше), но ближе к идее «команда агентов».

Полный цикл «аналитик + ресерчер + редактор + критик» из урока можно нарастить позже отдельными вызовами — сначала освой этот вариант.

---

## Урок 4 модуля 4 — кнопка «Тренды»

1. После `/start` внизу появится клавиатура с **«📈 Тренды»** (то же самое — команда **`/trends`**).  
2. Бот выдаст **3 темы** под твою нишу и Tone of Voice.  
3. Под сообщением — кнопки **«Пост по теме 1…3»**: нажми одну — придёт готовый пост.  
4. Опционально в `.env`: **`NICHE_HINT`** — одна строка, о чём канал (для точности тем).  
5. Опционально: **`TAVILY_API_KEY`** (регистрация на tavily.com) — тогда к темам подмешиваются короткие выдержки из поиска в интернете. Без ключа темы всё равно работают за счёт модели и твоего `tone-of-voice.md`.

**Домашка платформы:** три темы через эту функцию + один пост по одной из тем — сделай скрин переписки для сдачи.

## Расширения по смыслу урока М4 (после домашки)

| Кнопка / команда | Что делает |
|------------------|------------|
| 🎯 Идеи заголовков / `/hooks` | 5 заголовков под последнюю тему из чата или нишу |
| 🗣 Боли аудитории / `/audience` | Разбор болей по `tone-of-voice.md` |
| ✂️ Критика поста / `/critique` | Редакторская критика **последнего** ответа бота |
| 🖼 Картинка к посту / `/image` | Обложка: сначала `IMAGE_MODEL` на OpenRouter, иначе Pollinations |

**Поиск трендов сильнее:** в Railway добавь `TAVILY_*` из `.env.example` (в т.ч. `TAVILY_SOCIAL_EXTRA=1` для второго запроса «соцсети»).

**Skill для Cursor (IDE):** `.cursor/skills/copywriter-bot-m4/SKILL.md` — чтобы при правках кода агент помнил архитектуру.

## Если не работает

| Симптом | Что проверить |
|--------|----------------|
| Бот молчит | Запущен ли `npm start`, верный ли токен |
| Ошибка 401 | Неверный `CHAT_API_KEY` |
| Ошибка модели | Другое имя в `CHAT_MODEL` под твой провайдер |
| Странный стиль | Обнови `tone-of-voice.md`, потом `/reloadstyle` |
| Тренды не парсятся | Смени `CHAT_MODEL` на модель с устойчивым JSON; проверь лимиты API |
| Tavily не работает | Ключ `TAVILY_API_KEY` или оставь пустым — темы без веб-поиска |

Node.js нужен **версии 18+** (как в основном backend-проекте).
