# BACKEND-PLAN

## Цель
Сделать рабочий backend для Mini App и сайта, чтобы записи и заявки не терялись.

---

## Шаг 0 (домашка урока 4): сначала план — потом стройка

**Идея:** как с `brief.md` для сайта — сначала понятно *что* храним и *кто* что видит, потом подключаем инструменты.

### Что уже решено для этого проекта
| Сущность | Зачем |
|----------|--------|
| **masters** | Кто принимает записи (slug, имя, Telegram). |
| **bookings** | Записи из Mini App: дата рождения, числа, архетип, откуда пришёл клиент. |
| **leads** | Заявки с лендинга: имя, контакт, сообщение. |

### Кто что может
- **Клиент** в приложении и на сайте — только **отправить** данные (POST).
- **Мастер / ты** — **смотреть** списки через защищённый запрос (админ-токен) или через админ-страницу.
- В Mini App мастер выбирается параметром **`?master=marina`** (позже можно расширить под мульти-мастеров и Supabase).

### Два «склада» данных (это нормально на этапе обучения)
1. **Сейчас в коде:** SQLite файл `backend/data/app.db` — сервер уже пишет сюда. Удобно для проверок на компьютере.
2. **Домашка шаг 1:** такие же таблицы созданы в **Supabase** скриптом `supabase/schema.sql` — облачный фундамент. Подключение приложения к Supabase (API, ключи, RLS) — **следующий этап** урока (шаг 2+), не обязателен для сдачи пунктов 0–1.

## Шаг 1 (домашка): база в Supabase

- Пошаговая инструкция для новичка: **`SUPABASE-ШАГ1.md`**
- Готовый SQL: **`supabase/schema.sql`**
- После выполнения в **Table Editor** должны быть таблицы `masters`, `bookings`, `leads` и строка `marina` в `masters`.

### Что дальше (после шага 1, для справки)
- Включить **Row Level Security (RLS)** и политики: кто может читать/писать строки (например, только свой `master_slug`).
- Backend или Edge Functions ходят в Supabase с **service role** или через проверенный API — чтобы ключи не светились во фронте.

---

## API (текущая реализация на Node + SQLite)

- `GET /api/health` — сервер жив.
- `GET /api/masters/:slug` — данные мастера.
- `POST /api/bookings` — запись из Mini App.
- `GET /api/bookings` — список записей (заголовок `x-admin-token`).
- `POST /api/leads` — заявка с `index.html`.
- `GET /api/leads` — список заявок (`x-admin-token`).

## Технологии сейчас
- Node.js + Express
- SQLite (`backend/data/app.db`)
- CORS + JSON
