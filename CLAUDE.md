# Правила работы с проектом — Цифровой скрининг личности

## Кто я

Меня зовут Марина Филимонова. Я — цифровой скринолог, консультант по самопознанию через дату рождения.

Помогаю людям разобраться в себе и своей жизни через анализ числового кода: почему не складываются отношения, в чём сильные стороны, где теряется ресурс и почему повторяются одни и те же сценарии.

Это не про "предсказания" — про понимание своей природы, чтобы перестать идти против себя и начать опираться на реальные качества и возможности.

Я не программист. Я работаю в VS Code + Claude Code в стиле вайбкодинга — описываю что хочу словами, Claude пишет код.

## Мой проект

Онлайн-присутствие цифрового скринолога: визитка + лендинг + квиз. Детали — в project.md.

## Стиль дизайна

- Тёмные, глубокие цвета (фиолетовый, тёмно-синий, чёрный с акцентами)
- Премиальный, минималистичный вид
- Плавные переходы и мягкие тени
- Шрифты: элегантные, читаемые (serif для заголовков, sans-serif для текста)
- Никакого кричащего дизайна, никакого яркого розового или агрессивных цветов
- Атмосфера: доверие, спокойствие, глубина

## Технологии

- HTML + CSS (чистый, без фреймворков) — основа всего
- Никакого JavaScript-фреймворка (React, Vue и т.д.) — слишком сложно
- GitHub Pages — хостинг, всё должно работать как статичный сайт
- Файлы называть по-английски, без пробелов

## Что нельзя делать

- Не добавлять сложный код, который я не смогу понять
- Не использовать npm, node_modules и прочие установщики пакетов
- Не удалять существующие файлы без моего разрешения
- Не менять цветовую схему без явной просьбы
- Не добавлять платные сервисы или API без предупреждения
- Не писать комментарии в коде на английском — только на русском

## Как со мной работать

- Объясняй что делаешь простыми словами, без технического жаргона
- Если что-то сломается — скажи что случилось и как это исправить
- Всегда показывай результат (ссылку на GitHub Pages, если изменения отправлены)
- Спрашивай перед тем, как что-то удалять или сильно менять
